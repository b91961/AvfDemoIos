Visit Programmableweb.com and search for JSON-P data sources. Why would we need JSON-P, specifically?

JSON stands for JavaScript Object Notation. JSON-P stands for JSON with Padding. JSON is a lightweight text-based standard designed for human-readable data interchange. Instead of using AJAX, JSONP injects dynamic (script) tags to the document. It is a known fact that we can include scripts from other domains using the script tag. So in turn this allows for cross-domain calls.

Based on the JSON-P search results: what data resource(s) do you think might pair well with which types of Native device features?

JSON-P is primarily used in creating dynamic web site plugins like AdSense and badges, which are used in multiple sites dynamically to provide extensions. Some other data resources that might pair well with Native device features would be: Instagram with the camera, Dropbox with file, Google Maps with Geolocation and Twitter with notifications.

What are some of the ways that you could change your HTML on your Demo app to be more accessible? (Remember, you can use the Section tag with jQuery, for instance)

My HTML is written to be very accessible. There is not much I would like to change but it could probably be cleaned up a little bit From all the changes it does tend to get a little messy. I feel like just on this project I started to really understand how to use the HTML tags to mistype my CSS by using parents then child's and so forth. It wouldn't hurt to change most of my (div) tags over to (section) tags either. I might do it for the next project. All in all I am very happy with how my HTML has turned out.

Visit a video service on the web (like YouTube) and search for "Screen Reader Demo" and "Mobile Screen Reader" - what are some of the common frustrations (or common support mechanisms) that you see among the examples that are given - how could you fix them (or use them)?

I watched some demo videos on both the screen reader and the mobile screen reader. Neither of them displayed any problem or frustrations with the multiple videos that I watched. From my viewpoint as a developer, the one problem that I could see being a frustration would be the designer not using alt tags. If you have a picture of a Facebook logo with no text for the screen reader to read AB blind person would only know that was Facebook by the screen reader reading the alt tab of the picture.

Devices: Please list two devices (and some of their features) that are made specifically to support Accessibility.

The two most common type of devices that are made to specifically support accessibility would be a computer and a cell phone. Some features that both of these devices would have from the factory, would be magnifier for the vision impaired, screen readers for the blind, on screen dictation for the deaf And also Braille keyboards for the blind. There are a lot more accessibility features which you could explore by going to the settings on your phone or PCM and selecting accessibility.

Mobile Apps: Please list two apps (either Android or iOS, or both) that are made specifically to support Accessibility.

LookTel money reader for iOS devices allows a blind person to count money by using the iPhone camera. The app recognizes US currency and the screen reader announces the amount of the bill. Color identifier for iOS uses the iPhone camera to identify colors for the blind. This could be very useful in everyday life while getting dressed or shoppingIt besides those couple uses it is a way for a blind person to be in touch with a sense they never had or lost.
